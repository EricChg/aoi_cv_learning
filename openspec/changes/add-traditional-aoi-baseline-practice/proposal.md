# Proposal: Add traditional AOI baseline practice

## Summary

新增一个第 1-2 周 CV 基础与工业图像处理的实战变更：用 OpenCV 完成传统 AOI baseline、相机标定与畸变校正、正常样本统计分析三组练习，帮助学习者建立从图像预处理到候选缺陷输出的工程直觉。

## Motivation

当前路线图已经列出传统 AOI、相机标定、正常样本波动分析三项任务，但缺少可直接实现的工程化拆解。这个变更把任务转成明确脚本、输入输出、验收标准和学习记录，让学习者不仅知道要学什么，还能跑通一个可复现实验。

## Scope

- 新增传统 AOI baseline 脚本：读取图片、ROI、光照归一化、阈值/边缘/连通域、输出缺陷候选区域。
- 新增相机标定与畸变校正练习：基于棋盘格图片估计内参、畸变参数，并输出校正结果。
- 新增正常样本统计分析练习：对正常样本生成均值图、方差图、差分图，观察正常波动对检测阈值的影响。
- 新增命令行参数、结果目录约定和最小示例说明。
- 新增验收场景，确保输出文件和关键指标可检查。

## Out of Scope

- 不引入深度学习模型训练。
- 不实现产线相机采集 SDK、PLC/MES 对接或实时服务。
- 不要求使用真实工业 AOI 数据集；可以先用本地样例图片、公开图或自采图片完成练习。
- 不做复杂 GUI，仅保留脚本和可视化输出。

## Affected Teams

- 学习者：获得可运行练习和结果解释模板。
- 课程/文档维护者：需要在路线图或 README 中链接该练习。
- 后续深度学习与异常检测模块：可复用 ROI、归一化、候选区域和正常波动分析思路。

## Rollback Plan

如果该练习设计过重或不适合当前阶段，可以回滚新增脚本、示例说明和 spec delta，仅保留路线图中原有任务描述。回滚不会影响后续深度学习章节，因为本变更不修改共享训练接口或已有数据格式。

## Success Criteria

- 学习者可以用一条命令对单张图片生成候选缺陷 bbox/overlay。
- 学习者可以用一组棋盘格图片得到 camera matrix、distortion coefficients 和校正前后对比图。
- 学习者可以用一组正常样本得到 mean/variance/diff 可视化，并解释阈值误报与正常波动的关系。
- 所有输出都有稳定目录结构，便于写入学习笔记或后续实验报告。
