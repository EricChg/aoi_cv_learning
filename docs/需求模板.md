# 需求模板

## AOI 需求澄清表

| 模块 | 字段 | 需填写内容 | 备注 |
| --- | --- | --- | --- |
| 产品与工序 | 检测对象 |  | 产品名称、型号、来料状态 |
| 产品与工序 | 检测面 |  | 正面、背面、侧面、内壁、焊点区等 |
| 产品与工序 | 材质 |  | 金属、塑料、玻璃、PCB、膜材等 |
| 产品与工序 | 尺寸 |  | 长宽高、检测区域尺寸、关键结构尺寸 |
| 产品与工序 | 运动方式 |  | 静止拍照、皮带线、转盘、机械臂取放等 |
| 缺陷定义 | 缺陷类别 |  | 划伤、脏污、异物、缺料、错位、毛刺、裂纹、色差、印刷不良等 |
| 缺陷定义 | 最小可检缺陷 |  | 最小长度、宽度、面积、灰度/颜色差异 |
| 缺陷定义 | OK/NG 判定规则 |  | 哪些情况必须判 NG，哪些可接受 |
| 缺陷定义 | 边界样本规则 |  | 临界缺陷、可返修缺陷、疑似缺陷如何处理 |
| 输出形态 | 判定结果 |  | OK/NG、缺陷类别、严重等级 |
| 输出形态 | 定位结果 |  | 是否需要 bbox、mask、中心点、轮廓 |
| 输出形态 | 测量结果 |  | 面积、长度、宽度、位置偏差、数量 |
| 输出形态 | 追溯信息 |  | 原图、结果图、批次、设备号、时间戳 |
| 指标 | 漏检率目标 |  | 按整体或按缺陷类别填写 |
| 指标 | 误检率目标 |  | 可按产品、图片、缺陷框统计 |
| 指标 | 模型指标 |  | 召回率、mAP、IoU、pixel/image AUROC、PRO 等 |
| 指标 | 节拍要求 |  | 单件处理时间、整线吞吐、最大延迟 |
| 工程约束 | 成像配置 |  | 相机数量、分辨率、曝光、镜头、光源 |
| 工程约束 | 触发方式 |  | 硬触发、软触发、PLC 信号、编码器同步 |
| 工程约束 | 部署硬件 |  | 工控机、CPU、GPU、NPU、内存、磁盘 |
| 工程约束 | 部署位置 |  | 产线边缘端、本地服务器、云端 |
| 工程约束 | 系统对接 |  | PLC、MES、HMI、扫码枪、数据库、报警设备 |
| 数据现状 | 正常样本数量 |  | 按产品型号、批次、设备、日期统计 |
| 数据现状 | 缺陷样本数量 |  | 按缺陷类别统计，说明是否类别不均衡 |
| 数据现状 | 标注粒度 |  | 图像级、框级、像素级、关键点、测量值 |
| 数据现状 | 历史数据 |  | 是否有历史误判图、复判结果、产线日志 |
| 验收与上线 | 验收方式 |  | 离线测试、试产验证、A/B 对比、人工复判 |
| 验收与上线 | 上线策略 |  | 灰度上线、只报警不拦截、自动拦截、回滚方案 |

## 示例

| 模块 | 字段 | 示例内容 | 备注 |
| --- | --- | --- | --- |
| 产品与工序 | 检测对象 | 手机中框 CNC 后外观件，型号 A12 | 同一型号先做 PoC |
| 产品与工序 | 检测面 | 外侧四边和正面倒角区域 | 需要多相机或旋转工位 |
| 产品与工序 | 材质 | 阳极氧化铝，深灰/银色两种颜色 | 颜色会影响光照方案 |
| 产品与工序 | 尺寸 | 约 155 mm x 75 mm，重点检测边缘 5 mm 区域 | ROI 可按产品定位后裁剪 |
| 产品与工序 | 运动方式 | 转盘工位静止拍照，PLC 触发 | 每件 4 个视角 |
| 缺陷定义 | 缺陷类别 | 划伤、磕碰、脏污、毛刺、色差 | 首期只做前 4 类 |
| 缺陷定义 | 最小可检缺陷 | 划伤长度 >= 0.3 mm，宽度 >= 0.05 mm；脏污面积 >= 0.2 mm² | 低于阈值不计入 NG |
| 缺陷定义 | OK/NG 判定规则 | 任意关键区域划伤/磕碰判 NG；非关键区域轻微脏污可判 OK | 关键区域需在图上标出 |
| 缺陷定义 | 边界样本规则 | 疑似脏污进入人工复判，不直接拦截 | 降低初期误杀 |
| 输出形态 | 判定结果 | 输出 OK/NG、缺陷类别、置信度、严重等级 | MES 只接收最终判定 |
| 输出形态 | 定位结果 | NG 图保存 bbox 和结果叠图 | 便于复判和回流训练 |
| 输出形态 | 测量结果 | 输出缺陷长度、面积、距边缘距离 | 用于严重等级判断 |
| 输出形态 | 追溯信息 | 保存原图、结果图、SN、工位号、时间戳、模型版本 | 保存 90 天 |
| 指标 | 漏检率目标 | 关键缺陷漏检率 <= 0.1% | 以人工复判为准 |
| 指标 | 误检率目标 | 误检率 <= 2% | 试产期可放宽到 5% |
| 指标 | 模型指标 | 划伤召回率 >= 98%，bbox IoU >= 0.5 | 离线验证集统计 |
| 指标 | 节拍要求 | 单件 4 张图总处理时间 <= 800 ms | 含前处理、推理、后处理 |
| 工程约束 | 成像配置 | 4 台 500 万像素面阵相机，条形光 + 同轴光 | 需现场打光验证 |
| 工程约束 | 触发方式 | PLC 硬触发，拍照完成后回传 OK/NG 信号 | 与现有节拍同步 |
| 工程约束 | 部署硬件 | 工控机 i7 CPU + RTX 4060，16 GB 内存 | 预留后续多模型空间 |
| 工程约束 | 部署位置 | 产线边缘端本地部署 | 不依赖公网 |
| 工程约束 | 系统对接 | PLC 拦截信号、MES 上传结果、HMI 展示 NG 图 | 需要定义接口协议 |
| 数据现状 | 正常样本数量 | 正常图 12000 张，覆盖 3 个批次 | 需补充银色产品 |
| 数据现状 | 缺陷样本数量 | 划伤 800、磕碰 350、脏污 1200、毛刺 200 | 毛刺样本偏少 |
| 数据现状 | 标注粒度 | bbox 标注；关键缺陷补充 mask 标注 200 张 | 先检测，后续再分割 |
| 数据现状 | 历史数据 | 有人工误判图 500 张和试产日志 | 可作为 hard negative |
| 验收与上线 | 验收方式 | 离线测试 3000 张 + 产线试跑 3 天 + 人工复判 | 输出每日统计报表 |
| 验收与上线 | 上线策略 | 第一周只报警不拦截；稳定后对关键缺陷自动拦截 | 保留模型回滚开关 |
